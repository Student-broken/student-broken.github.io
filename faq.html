Sécurité du Classement

Le système de classement a été développé selon une approche de "privacy-by-design" (protection de la vie privée dès la conception). Voici une explication technique des mesures de sécurité en place.

Mon nom ou identifiant personnel sera-t-il rendu public ?

Absolument pas. Le classement utilise exclusivement un identifiant anonyme généré aléatoirement : user-random. Votre code basé sur votre nom (userCode) est une clé utilisée uniquement par le système interne pour enregistrer vos notes et n'est jamais transmis à aucun utilisateur pour l'affichage du classement.

Comment le système identifie-t-il mon rang s'il est anonyme ?

Ce processus se déroule entièrement de votre côté (client-side). Le serveur envoie une liste publique contenant uniquement les identifiants user-random et les notes. Votre navigateur, qui conserve de manière sécurisée votre propre user-random, scanne cette liste. Lorsqu'il trouve une correspondance, il applique dynamiquement l'étiquette "Vous" sur cette entrée. Cette identification est privée et n'a lieu que sur votre appareil.

Est-il possible de manipuler le serveur pour exposer les noms ?

Non. Le point d'accès public de l'API (la fonction doGet) qui fournit les données du classement est programmé côté serveur pour exclure systématiquement la colonne contenant le userCode de sa réponse. Il est donc techniquement impossible pour un utilisateur de demander ou de recevoir cette donnée privée.

Comment mes données sont-elles protégées lors de l'envoi ?

Toutes les communications entre votre navigateur et le serveur Google Apps Script sont chiffrées via HTTPS. C'est le standard de l'industrie pour la sécurité de la couche de transport, garantissant que vos informations de notes et vos identifiants ne peuvent pas être interceptés pendant leur transit.

Pourquoi stocker le userCode dans la base de données ?

Le userCode sert de clé stable et permanente pour votre enregistrement de données. Sa seule fonction est de permettre au système de mettre à jour vos notes avec précision au fil du temps, même si vous changez d'appareil ou effacez le cache de votre navigateur (ce qui générerait un nouvel user-random). Cette base de données est privée et totalement inaccessible depuis le classement public.
